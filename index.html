    Logic Explanation

    This application stores loot items inside an array of objects where each object contains a name and value property. When the user adds loot, an object is pushed into the array, allowing the data to persist across button clicks.

    A traditional for loop is used in the renderLoot function to dynamically display each loot item in the interface and to calculate the running total of all loot values. Another loop is used in the splitLoot function to compute the final total before dividing it among party members.

    Conditional logic is used to validate user inputs such as checking for empty loot names, invalid or negative values, missing party size, and cases where no loot exists. These conditions ensure the application handles edge cases and prevents incorrect calculations.
